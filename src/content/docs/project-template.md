# Project Page Template

Use this structure when we start or adopt a project.

## Summary
What the project is and why it exists.

## Current State
- status
- what's working
- what's blocked
- what is unclear

## Goal
What success looks like.

## Users / Audience
Who it is for.

## Scope
What is in and what is out.

## Architecture / Shape
High-level structure, tools, services, and moving parts.

## Workflow
How work happens in this project.

## Active Risks
What could break, stall, or create rework.

## Decisions
Important decisions made so far.

## Next Steps
Immediate next actions.

## References
Links, repos, docs, dashboards, files.

## Good prompts
- Give me a project pulse for this project.
- Turn this into a proper project page.
- Update this project page with today’s changes.
